| 🔴 极度危险 | `src/ui/views/dashboard/index.tsx` (Line 80) | **[闭包陷阱与过期状态]**: 当 `Dashboard` 组件被卸载时 (Unmounted)，`useEffect` 的清理函数中如果 `pendingLogs` 恰好有残留记录，它会强行调用 `setLogs`。<br>**代码片段**: `setLogs(prev => [...pendingLogs, ...prev].slice(0, 4));`<br>这会导致在已销毁的组件上强行触发 React 状态更新，不仅会引发内存溢出警告，还会打断正常的生命周期喵！ | 既然组件都要被销毁了，UI 也不需要刷新啦喵~ 建议在组件顶层挂载一个 `isMounted` ref，如果卸载了直接跳过 flush 逻辑喵！ |
| 🔴 极度危险 | `src/modules/batch/engine/BatchEngine.ts` (Line 137) | **[异步竞态与卸载冲突]**: `execute` 方法中有一段异步请求 `await handler.estimate()`。如果在等待期间外部调用了 `stop()`，`stopSignal` 确实会变成 true 并把现有队列的任务标记为 skipped。但是！等 `estimate` Promise 返回后，代码直接用**新的 tasks** 覆盖了队列，导致新任务的状态全是 pending。接着循环因为检查到 `stopSignal` 直接 break，导致这些新任务一直卡在 pending 状态，UI 永远不往下走，彻底脱节喵！<br>**代码片段**: `this.queue.tasks = tasks;` | 强烈建议在 `const tasks = await handler.estimate();` 之后，再次检查一次 `if (this.stopSignal) return;` 保护一下队列状态喵！ |
