| 🔴 极度危险 | `src/modules/batch/engine/BatchEngine.ts` (Line 214) | **[链路追踪]**: 当触发 `stop()` 动作时，方法内部会调用 `setTimeout(() => this.clearQueue(), 500);`。如果在这 500ms 内，用户或者程序立刻调用了 `execute()` 加入了新的任务，新任务会被瞬间清空掉喵！这会导致任务静默消失（Zombie Task）~<br>**代码片段**: `setTimeout(() => this.clearQueue(), 500);` | 建议在 `clearQueue` 内增加对 `stopSignal` 或当前 `queue.isRunning` 的二次校验，或者直接在 `stop()` 内同步清理队列状态，避免异步清理带来的竞态漏洞喵~ |

| 🟠 性能瓶颈 | `src/data/sync/SyncService.ts` (Line 235) | **[链路追踪]**: `getRemoteStatus` 仅仅为了获取一个 `meta.lastModified` 时间戳，就使用 `await response.json()` 将一整个同步文件解析到内存里。如果用户的云端备份文件很大（比如好几十MB的聊天记录），这会严重阻塞主线程，导致 UI 卡顿甚至崩溃喵！<br>**代码片段**: `const dump = await response.json() as ChatDataDump;` | 虽然酒馆不支持 HEAD 请求，但我们可以使用流式读取 (`response.body.getReader()`) 或者正则提取只读前部几十 KB，找到 meta 信息后就尽早 abort 这个 fetch 请求喵！不要把整个 JSON 读进内存~ |
