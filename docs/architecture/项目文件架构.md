# Engram 项目文件架构

> 基于 **React + Vite + TypeScript** + **Tailwind CSS**
> **Version**: v0.9.9 (Layered Modular Architecture)

---

## 1. 顶层架构概览 (High-Level Overview)

```
src/
├── core/                  # [L0] 核心层：基础类型、事件总线、工具库
├── config/                # [L1] 配置层：设置管理 (SettingsManager)
├── data/                  # [L2] 数据层：IndexedDB 定义与聊天管理
├── integrations/          # [L3] 集成层：外部系统适配 (SillyTavern, LLM)
│   ├── tavern/            # 宿主适配 (含 WorldBook 深度集成)
│   └── llm/               # LLM 调用适配
├── modules/               # [L4] 模块层：核心业务逻辑单元
│   ├── batch/             # 批处理系统
│   ├── memory/            # 记忆管理 (Summarizer, Extractor)
│   ├── preprocessing/     # 输入预处理 (Preprocessing)
│   ├── rag/               # Graph RAG 检索 (Brain, Embedding)
│   ├── search/            # 搜索服务 (Vector Search)
│   └── workflow/          # 任务流编排引擎
│       ├── core/          # 引擎核心 (Engine, Step)
│       ├── definitions/   # 预定义工作流
│       └── steps/         # 原子步骤库
├── state/                 # [L5] 状态层：Zustand Store
└── ui/                    # [L6] 表现层：React 组件与视图
```

## 2. 详细目录结构 (Detailed Structure)

### 2.1 [L0] Core Layer (`src/core`)
系统最底层的契约，不依赖任何其他层级。

```
src/core/
├── events/                # 事件总线定义
├── logger/                # 日志系统
├── types/                 # 全局类型定义
├── updater/               # 更新检查器
├── utils/                 # 通用工具函数 (JSONParser, Short UUID)
└── KeyboardManager.ts     # 全局快捷键管理
```

### 2.2 [L1] Config Layer (`src/config`)
负责配置的定义、默认值和读写。

```
src/config/
├── settings.ts            # SettingsManager (对接 extension_settings)
└── types/                 # 配置接口定义
    ├── defaults.ts        # 默认值聚合
    ├── rag.ts             # RAG 相关配置
    ├── prompt.ts          # 提示词模板配置
    └── ...
```

### 2.3 [L2] Data Layer (`src/data`)
负责数据的本地持久化存储。

```
src/data/
├── db.ts                  # Dexie 数据库定义 (EngramDB)
├── ChatManager.ts         # 聊天会话生命周期管理
├── cleanup/               # 数据清理服务
├── repositories/          # 数据仓储层 (Repository Pattern)
└── sync/                  # 数据同步服务
```

### 2.4 [L3] Integrations Layer (`src/integrations`)
防腐层，隔离外部依赖变换。

```
src/integrations/
├── tavern/                # SillyTavern 宿主适配
│   ├── bridge.ts          # Window 对象桥接 (ST 交互核心)
│   ├── context.ts         # 上下文获取 (Chat, Character)
│   ├── macros.ts          # 宏注册服务 (Register Macros)
│   ├── events.ts          # 酒馆事件监听
│   ├── api/               # 宿主 API 封装 (Message, Extension)
│   └── worldbook/         # 世界书深度集成模块
│       ├── adapter.ts     # 适配器
│       ├── crud.ts        # 增删改查逻辑
│       ├── engram.ts      # Engram 专用世界书逻辑
│       ├── scanner.ts     # 扫描器
│       └── metrics.ts     # 指标统计
└── llm/                   # LLM 服务适配
    ├── Adapter.ts         # LLM 统一调用适配器
    └── SystemPrompts/     # 内置提示词文件 (.md)
```

### 2.5 [L4] Modules Layer (`src/modules`)
核心业务逻辑，按功能垂直切分。

#### Workflow (工作流引擎)
负责编排和执行复杂的业务逻辑链。
```
src/modules/workflow/
├── core/                  # 新版工作流引擎
│   ├── WorkflowEngine.ts  # 执行引擎 (支持跳转、终止)
│   ├── JobContext.ts      # 任务上下文定义
│   └── Step.ts            # 步骤接口定义
├── definitions/           # 预定义工作流
│   ├── EntityWorkflow.ts  # 实体提取流
│   ├── SummaryWorkflow.ts # 总结生成流
│   └── ...
└── steps/                 # 原子步骤库
    ├── context/           # 获取上下文 (GetChatHistory)
    ├── execution/         # 执行操作 (LLMRequest)
    ├── interaction/       # 用户交互 (UserReview)
    ├── persistence/       # 持久化 (SaveEntity)
    ├── processing/        # 数据处理 (ParseJSON)
    └── rag/               # RAG 检索管线 (VectorRetrieveStep 等)
```

#### RAG & Search (检索与搜索)
```
src/modules/
├── rag/                   # RAG 核心
│   ├── retrieval/         # 召回策略 (BrainRecallCache, Hybrid)
│   ├── embedding/         # 向量化服务 (EmbeddingService)
│   └── injection/         # 上下文注入 (Injector)
└── search/                # 搜索基础服务
    ├── SearchService.ts   # 搜索服务入口
    └── adapters/          # 搜索引擎适配
```

#### Memory & Processing (记忆与处理)
```
src/modules/
├── memory/                # 记忆管理
│   ├── Summarizer.ts      # 剧情总结生成服务
│   ├── EntityExtractor.ts # 实体提取服务 (Facade)
│   ├── EventTrimmer.ts    # 记忆精简逻辑
│   └── ...
├── preprocessing/         # 输入预处理
│   ├── Preprocessor.ts    # 预处理主控
└── batch/                 # 批处理系统
    ├── engine/            # 批处理调度底层 (BatchEngine)
    ├── tasks/             # 预定义子任务处理器 (HistoryTask, 等)
    └── types.ts           # 批处理数据类型规范
```

### 2.6 [L5] State Layer (`src/state`)
UI 状态管理，作为 MVC 中的 Controller。

```
src/state/
├── memoryStore.ts         # 核心记忆状态 (CRUD, 过滤, 排序)，包括 eventSlice 下的 Agentic RAG 高阶提取函数 (getAgenticIndex 等)
├── devLogStore.ts         # 开发日志状态
├── themeStore.ts          # 主题与外观状态
└── index.ts               # 状态导出
```

### 2.7 [L6] UI Layer (`src/ui`)
React 视图与交互组件。

```
src/ui/
├── components/            # 通用组件库
│   ├── core/              # 核心基础组件 (Button, Input, SliderField)
│   ├── display/           # 展示类组件 (Card, Badge)
│   ├── feedback/          # 反馈类组件 (Modal, Toast)
│   └── layout/            # 布局组件 (MasterDetail)
├── hooks/                 # 业务 Hooks
│   ├── useDashboardData.ts # 仪表盘数据聚合
│   └── ...
└── views/                 # 页面视图
    ├── api-presets/       # API 与提示词配置
    ├── dashboard/         # 概览仪表盘
    ├── dev-log/           # 开发者日志
    ├── docs/              # 帮助文档
    ├── memory-stream/     # 记忆流可视化 (核心视图)
    ├── processing/        # 处理面板 (Review Modal)
    ├── quick-panel/       # 快速面板
    ├── review/            # 审核面板
    └── settings/          # 设置面板
```

### `core/` - 核心层
整个系统的基石，定义了数据流动的基本契约。
- **Events**: 定义了应用内通信的事件类型。
- **Types**: 包含跨层级使用的通用接口定义。

### `config/` - 配置层
负责应用配置的定义、默认值和持久化读写。
- **SettingsManager**: 统一管理扩展配置的读写，对接 SillyTavern 的 `extensionSettings`。

### `data/` - 数据层
负责数据的本地持久化。
- **EngramDB**: 基于 Dexie.js 的 IndexedDB 封装。
- **ChatManager**: 管理当前聊天会话的状态和元数据。

### `integrations/` - 集成层
充当防腐层 (Anti-Corruption Layer)，隔离外部系统变化对核心逻辑的影响。
- **Tavern**: 封装 SillyTavern 的全局变量和 API 调用。
- **LLM**: 封装不同 LLM 后端的调用差异，提供统一的 `generate` 接口。

### `modules/` - 模块层 (业务核心)
包含独立的功能模块，每个模块由 `index.ts` 导出公共 API。

| 模块 | 职责 | 关键导出 |
|------|------|----------|
| **rag** | 检索增强生成 | `Retriever`, `VectorService`, `Reranker` |
| **summarizer** | 剧情总结 | `SummarizerService`, `SummaryWorker` |
| **preprocessing** | 输入预处理 | `Preprocessor` |
| **graph** | 实体与图谱 | `EntityBuilder`, `GraphService` |
| **batch** | 批处理任务 (含健壮性防坠毁) | `BatchProcessor`, `BatchQueue` |

### `state/` - 状态层
连接业务逻辑与 UI，使用 Zustand 实现响应式状态管理。
- **memoryStore**: 应用的核心状态仓库，UI 组件通过 Hooks 订阅此处的变更。

## 🏗️ 关键模块说明

### Workflow Engine
- **定位**: 系统的逻辑中枢，逐步替代散落的业务逻辑。
- **特点**: 支持步骤复用、可视化追踪、错误处理、用户交互暂停，以及基于 action (finish, abort, jump) 的控制流流转。
- **核心文件**: `WorkflowEngine.ts`, `Step.ts`

### Tavern Integration (Host Adapter)
- **定位**: `src/integrations/tavern` 扮演整个 Engram 系统与 SillyTavern 宿主的防腐层。
- **核心组件**:
  - `worldbook`: 不仅仅是 API 包装，更是 Engram 与 SillyTavern 世界书系统的深度同步模块，管理 Engram 生成的词条（增删改查），并同步回 SillyTavern，确保数据一致性。
  - `Adapter.ts` (`src/integrations/llm/Adapter.ts`): 通过 `getTavernHelper()` 封装底层原生的 `window.TavernHelper` 调用，避免直接访问全局变量可能引发的奔溃，确保容错与模块隔离。

### BrainRecallCache (RAG)
- **定位**: `src/modules/rag/retrieval/BrainRecallCache.ts`
- **功能**: 实现了类脑记忆机制（短期记忆、工作记忆、遗忘曲线），负责管理 RAG 召回的上下文窗口。
