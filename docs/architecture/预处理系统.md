# Engram 输入预处理系统 V0.8

> **创建日期**: 2026-01-11
> **版本**: V0.8.5 (Updated Path)
> **状态**: ✅ 已实现

## 1. 功能概述

输入预处理系统在用户发送消息时自动进行预处理，通过 LLM 增强用户输入，支持多种功能模式。

### 1.1 核心理念

**输入预处理 = 动态提示词模板 + 统一输出解析**

- 所有预处理模板统一使用 `preprocessing` 分类
- 输出统一用 `<output>` 标签包裹
- Query 增强模式额外使用 `<query>` 标签
- 系统统一捕获和处理输出
- 支持直接注入 (Skip To Injection) 机制，可以跳过输入替换，直接将内容作为 AI 消息注入

### 1.2 内置功能模式

| 模式 | 用途 | 输出 |
|------|------|------|
| **Query 增强** | 扩展指代词，优化 RAG 检索 | `<query>` + `<output>` |
| **剧情编排** | 生成导演指令框架 | `<output>` |
| **描写增强** | 补充细节描写和环境氛围 | `<output>` |
| **自定义** | 用户创建的任意预处理 | `<output>` |

## 2. 系统架构

### 2.1 核心组件

```
src/modules/preprocessing/
├── Preprocessor.ts       # 预处理核心服务
└── types.ts              # 类型定义

src/modules/rag/injection/
└── Injector.ts           # 事件监听和注入
```

### 2.2 事件驱动流程

```
用户点击发送
    ↓
GENERATION_STARTED
    ↓
处理斜杠命令
    ↓
GENERATION_AFTER_COMMANDS ← 🎯 Engram 预处理入口
    ↓ (await 阻塞)
Preprocessor.process(userInput)
    ↓
WorkflowEngine.run(createPreprocessWorkflow())
    │
    ├── [若存在 Skip To Injection]
    │   ├── 调用 injectMessage 将预处理输出作为 AI 消息注入
    │   ├── 拦截并清空当前用户输入 (中止本次用户交互)
    │   └── 结束预处理流程
    │
    └── [常规分支：提取并解析]
        ├── 提取 <output>...</output>
        ├── 提取 <query>...</query> (如有)
        ├── 提取并解析 <recall_decision> (解析出 agenticRecalls 数组)
        └── 拦截 UserReview 进行人工审核判定
    ↓
修改 lastMessage.mes = result.output 并在 Injector 中回写 Textarea 或覆盖 Chat
    ↓
酒馆继续正常生成
```

### 2.3 阻塞机制

关键发现：SillyTavern 的 `eventSource.emit()` 会 `await` 所有事件处理器。

```javascript
// SillyTavern script.js
await eventSource.emit(event_types.GENERATION_AFTER_COMMANDS, type, params, dryRun);
```

因此只需确保 Engram 的事件处理器是 `async` 函数即可实现阻塞。

## 3. 数据结构

### 3.1 预处理配置 (来自 config/types/data_processing.ts)

```typescript
export interface PreprocessingConfig {
    enabled: boolean;        // 是否启用预处理
    templateId: string;      // 当前使用的模板 ID
    preview: boolean;        // 是否在应用前进行预览 (预览修订)
}
```

### 3.2 预处理结果

```typescript
export interface PreprocessingResult {
    success: boolean;        // 是否成功
    output: string | null;   // <output> 内容 - 注入到用户消息 (如果直接注入，可能为空)
    query: string | null;    // <query> 内容 - RAG 检索关键词 (Query 增强模式)
    rawOutput: string;       // 原始 LLM 输出 (已清理 <think>)
    recalledIds?: string[];  // 召回的事件 ID
    agenticRecalls?: AgenticRecall[]; // Agentic RAG: <recall_decision> 解析结果
    processingTime: number;  // 处理耗时 (ms)
    error?: string;          // 错误信息
}
```

## 4. 统一输出格式

### 4.1 标准模板输出

```xml
<think>
... 思考过程（会被删除）...
</think>

<output>
... 实际输出内容（会被捕获并注入用户消息）...
</output>
```

### 4.2 Query 增强模式

```xml
<think>
... 分析用户意图 ...
</think>

<query>
关键词1, 关键词2, 相关概念
</query>

<output>
[用户意图总结和上下文补充]
</output>
```

- `<query>` 内容 → 发送给 RAG 系统进行向量检索
- `<output>` 内容 → 注入用户消息

### 4.3 预处理直接注入 (Skip to Injection)

如果 Workflow 判断需要跳过直接注入（例如预览修订模式并选择作为 AI 消息回复），`skipToInjection` 标记会被设置为 `true`。此时系统将直接调用 `injectMessage('char', output)` 并在返回结果中将 `output` 清空，避免对用户输入进行错误覆盖。

## 5. 提示词模板

### 5.1 内置模板

| 模板 | 默认状态 |
|------|---------|
| Query 增强 | 启用 |
| 剧情编排 | 禁用 |
| 描写增强 | 禁用 |

### 5.2 支持的宏变量

| 宏 | 说明 |
|------|------|
| `{{worldbookContext}}` | 世界书激活内容 |
| `{{chatHistory}}` | 最近对话历史 |
| `{{userInput}}` | 用户本轮输入 |
| `{{char}}` | 角色名 |
| `{{user}}` | 用户名 |
| `{{context}}` | 完整上下文 |

### 5.3 自定义模板

用户可以在 **API 配置 (API Presets) → 提示词模板** 中：
- 创建新的预处理模板 (选择 `preprocessing` 分类)
- 编辑内置模板的系统提示词和用户提示词
- 导入/导出模板配置

## 6. 使用指南

### 6.1 启用预处理

1. 打开 Engram 面板
2. 进入 **API 配置 (API Presets) → 提示词模板**
3. 找到 "Query 增强" 或其他预处理模板
4. 点击电源图标启用模板
5. 在 **处理** 页面或 **快捷面板 (QuickPanel)** 开启 "输入预处理" 开关。

### 6.2 切换功能模式

在快捷面板 (QuickPanel) 中，可以直接在可用的预处理模式列表中进行快速切换。快捷面板会实时监听并与底层 `preprocessingConfig` 和全局 `recallConfig` 保持同步。

### 6.3 预览与修订

在快捷面板中可开启 **"预览修订" (preview)** 开关。启用后，生成内容将会进入预览模式（对应 `UserReview` 等交互拦截节点），允许用户进行审核、修改或直接作为 AI 角色消息进行注入 (`skipToInjection`)。

## 7. 与 RAG 的集成

### 7.1 Query 增强与 Agentic 召回联动

```
用户输入 "我们之前约定的那件事"
    ↓
Preprocessor 调用 WorkflowEngine.run (使用 query_enhance 或 agentic_recall 模板)
    ↓
LLM 输出 (提取意图或裁决 ID):
<query>约定, 承诺, 之前的对话</query>
<recall_decision> { "recalls": [...] } </recall_decision>
<output>用户提及之前的约定，需要回顾相关记忆</output>
    ↓
Workflow 节点执行 (ExtractTags)
├── query: "约定, 承诺, 之前的对话"
├── output: "用户提及之前的约定，需要回顾相关记忆"
└── agenticRecalls: 提取出的 AgenticRecall[] 数组
    ↓
Injector.ts 拦截提取后的结果：
如果检测到启用 Agentic RAG 并且 agenticRecalls 存在，触发 Retriever.agenticSearch 直通 ID 检索。
否则降级 fallback 至常规向量检索。
    ↓
MacroService 缓存更新 (如 refreshCacheWithNodes)
```

### 7.2 非 Query 模式

直接将 `<output>` 内容替换原始用户消息：

```
用户输入 → LLM (plot_director) → <output>导演指令</output>
    ↓
lastMessage.mes = "导演指令"
    ↓
酒馆继续生成 (使用增强后的输入)
```

## 8. 注意事项

### 8.1 跳过条件

以下情况预处理会自动跳过：
- 预处理未启用
- `dryRun` 模式 (预览/计算 Token)
- `regenerate` 重新生成
- `swipe` 切换回复
- `quiet` 静默生成
- `impersonate` 代入模式

### 8.2 取消请求控制

通过 `notificationService` 提供的运行中 toast 通知，用户可以直接点击取消请求。此时系统将通过调用 `StopGeneration.abort()` 请求终止流。

### 8.3 错误处理

预处理失败时会记录日志，返回带有错误信息的失败结果。如果由于用户取消操作引发 `UserCancelled`，系统也将安全退出预处理而不引发整个生成管线的崩溃。
