## <AlertTriangle className="inline w-5 h-5 mb-1 text-yellow-500" /> 运行与连接问题

### Q1：插件无响应、界面空白或按钮失效？
- **优先检查**：运行日志是否有 `ERROR`。
- **常见原因**：IndexedDB 初始化失败、宿主事件未挂载、版本不兼容。
- **处理建议**：刷新页面重试；必要时清理异常数据库后重建。

### Q2：API 404 / Connection Refused / 超时？
- **原因**：模型服务地址、端口、协议或 Key 配置错误。
- **处理建议**：
  1. 检查 Base URL（通常需要包含 `/v1`）。
  2. 检查本地服务跨域策略（如 Ollama 的 `OLLAMA_ORIGINS`）。
  3. 某些网关要求 Key 非空，即使本地也需填写占位值。

### Q3：更新插件时出现 `Forbidden` / `EACCES`？
- **原因**：SillyTavern 进程权限不足，无法覆盖旧文件。
- **处理建议**：关闭占用进程后重新安装，或调整目录读写权限。

## <Workflow className="inline w-5 h-5 mb-1 text-primary" /> 处理链路问题（总结 / 提取 / 精简）

### Q4：自动总结没触发？
- **先看模型日志是否有请求**：
  - 没请求：通常是触发条件没达到（间隔、缓冲、开关）。
  - 有请求但失败：看返回码与解析错误。
- **再看运行状态**：若上一轮任务卡住，串行保护会阻止新任务进入。

### Q5：总结结果质量差或重复？
- **原因候选**：模型能力不足、提示词不匹配、历史窗口设置不合理。
- **处理建议**：
  1. 更换更稳定模型。
  2. 调整 Summary 模板。
  3. 拉大间隔并校正缓冲区，减少噪声输入。

### Q6：Trimmer 合并后“感觉记忆丢了”？
- **解释**：Trimmer 是层级压缩，不是简单删除。
- **建议**：
  - 对关键节点使用 <Lock className="inline w-4 h-4 mb-1" /> 锁定保护。
  - 在记忆流中同时检查低层与高层摘要是否语义一致。

## <Search className="inline w-5 h-5 mb-1 text-primary" /> 检索与召回问题（RAG）

### Q7：明明聊过，为什么没召回？
- **检查顺序**：
  1. 召回日志中的 query 是否合理。
  2. 命中得分是否过低。
  3. 是否被上下文窗口截断或模型忽略。

### Q8：我改了记忆文本，但搜不到新词？
- **原因**：向量索引未更新。
- **处理**：对条目执行 **Re-embed**。

### Q9：开启 RAG 后明显变慢？
- **原因**：Embedding + Rerank 会增加请求链路。
- **优化**：
  - 优先本地 Embedding。
  - 在可接受精度下关闭或降频 Rerank。

## <WandSparkles className="inline w-5 h-5 mb-1 text-primary" /> 预处理问题

### Q10：预处理没有生效？
- **确认项**：
  1. 已启用预处理总开关与自动触发。
  2. 当前模板属于 `preprocessing` 分类。
  3. 模板输出包含 `<output>` 标签。

### Q11：Query 增强开了但召回仍然不准？
- **检查**：模型是否输出了 `<query>`，以及 query 是否过泛。
- **优化**：减少空泛词，增加实体名、地点名、事件锚点词。

## <Archive className="inline w-5 h-5 mb-1 text-primary" /> 生命周期与数据管理

### Q12：有些记忆“消失”了？
- **大概率**：进入归档态（绿灯），不再常驻显示。
- **说明**：被召回时仍可临时进入上下文，不等于删除。

### Q13：锁定（Lock）到底有什么用？
- **锁定实体**：防止自动归档。
- **锁定事件**：防止被 Trimmer 合并。
- **场景**：核心人设、主线转折、关键设定条款。

## <Clock className="inline w-5 h-5 mb-1 text-primary" /> 一条实用建议

遇到问题时按这三步执行，成功率最高：
1. 先看日志定位是“触发问题、模型问题还是检索问题”。
2. 只改一个参数并复测，避免多变量干扰。
3. 记录有效配置，形成你自己的稳定预设。
