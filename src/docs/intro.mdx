# 欢迎使用 Engram

> **Engram** 是 SillyTavern 的本地优先记忆操作系统：以 **Workflow** 编排任务、以 **IndexedDB** 持久化记忆、以 **宏注入** 安全接入上下文。

---

## 一句话理解系统

Engram 的核心链路是：**触发请求 → Service Facade → Workflow Steps → IndexedDB → Macro 注入到 LLM 请求**。

## 核心机制总览

### 1) 记忆沉淀（Consolidation）

- 达到楼层间隔后触发总结流程。
- 从历史中提取 `EventNode`（摘要）与 `EntityNode`（实体）。
- 支持后续 Trimmer 精简与层级化存储（Level 0/1+）。

### 2) 记忆召回（Retrieval）

- 检索路径由 `Retriever` 统一调度。
- 支持传统向量召回、混合召回与 Agentic RAG 直通。
- 召回结果进入缓存后参与宏注入，影响当前轮模型上下文。

### 3) 可见性策略（蓝灯/绿灯）

- 🔵 蓝灯：`is_archived=false`，常驻可见。
- 🟢 绿灯：`is_archived=true`，仅在被召回时临时进入上下文。

## 数据与隐私

- **Zero Server Dependency（核心存储）**：记忆数据存于浏览器 IndexedDB。
- **模型可插拔**：支持 OpenAI 兼容接口与本地模型服务。
- **增量处理**：只处理新数据，减少重复计算与 Token 浪费。
