# 欢迎使用 Engram

> **Engram** 是 SillyTavern 的本地优先记忆操作系统：以 **工作流** 编排任务、以 **本地数据库** 持久化记忆、以 **宏变量注入** 安全接入上下文。

---

## 一句话理解系统

Engram 的核心链路是：**触发请求 → 门面服务层 → 工作流步骤处理 → 本地数据库持久化 → 宏变量注入到大模型请求**。

## 核心机制总览

### 1) 记忆沉淀（存储与精简）

- 达到楼层间隔后触发自动总结流程。
- 从聊天历史中提取事件摘要与人物实体。
- 支持后续记忆精简（Trimmer）与层级化存储（基础事件 / 高层大纲）。

### 2) 记忆召回与宏变量注入

- 检索路径由基于工作流的统一检索器调度完成。
- **混合检索**: 支持向量匹配与重排序（Rerank）模型的联合打分。
- **主动直通检索 (Agentic RAG)**: 大模型作为裁判，直接提取指定目标记忆集合，彻底绕过传统向量阶段，精准推入类脑缓存机制中。
- 最终结果进入类脑缓存池后，统一通过 `{{engramIndex}}`（全量索引字典）或 `{{engramActiveEvents}}`（近期纯净活跃事件数组）聚合，暴露给宿主的提示词系统。

### 3) 可见性策略（蓝灯/绿灯）

- 🔵 蓝灯：`is_archived=false`，常驻可见。
- 🟢 绿灯：`is_archived=true`，仅在被召回时临时进入上下文。

## 数据与隐私

- **零服务端依赖**：记忆数据永远存于浏览器本地。
- **模型可插拔**：支持兼容 OpenAI 标准格式的各种接口与本地私有化部署的模型服务。
- **增量处理**：始终只处理最新聊天数据，减少重复计算与 Token 消耗。
